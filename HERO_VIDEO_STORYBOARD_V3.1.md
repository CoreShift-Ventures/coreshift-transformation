#### 2. Screen Recording (Week 3, Day 6-7)

**Tools:**
- **Mac:** Screen Studio (https://screen.studio) - **HIGHLY RECOMMENDED**
  - Auto-smooths cursor movement
  - Professional zoom/pan effects
  - Exports polished MP4
- **Windows:** OBS Studio or Camtasia
- **Cross-platform:** Loom (simpler but less polished)

**Recording Tips:**
- Record at **4K resolution** (3840×2160) if possible, downscale to 1080p for final
- Use **60fps** for smoother animations
- **Rehearse each shot 3-4 times** before final recording
- Record in **segments** (one shot at a time) - easier to edit
- Keep cursor movements **intentional and smooth**
- **Hide** browser UI, bookmarks, personal info

**Recording Plan:**
1. **Session 1:** Record Shot 1 (Lead Conversion - 3 segments)
2. **Session 2:** Record Shot 2 (Order Management - 3 segments)
3. **Session 3:** Record Shot 3 (Customer Success - 3 segments)
4. **Session 4:** Record Shot 4 (Closing - 1 segment)

#### 4. Optimization for Web (Week 3, Day 10)

**Export Settings:**
- **Resolution:** 1920×1080 (Full HD)
- **Frame Rate:** 60fps (or 30fps if file size too large)
- **Codec:** H.264 (MP4) - best compatibility
- **Bitrate:** 5-8 Mbps (balance quality/file size)
- **Audio:** AAC, 128kbps

**Target File Size:** <5MB
- If file >5MB, reduce bitrate or use 30fps

**Create WebM Version:**
- Use FFmpeg or Handbrake to create WebM version
- Often 30-50% smaller than MP4
- Modern browsers support WebM
- Fallback to MP4 for Safari

**Compression Tools:**
- Handbrake (free, excellent)
- FFmpeg (command-line)
- Adobe Media Encoder
- Online: CloudConvert

**Create Poster Image:**
- Export frame from 0:01 (after title appears)
- Save as JPEG or WebP
- Use as video thumbnail (shows before video loads)

**Test Playback:**
- Test autoplay/loop on all major browsers
- Test on mobile (iOS, Android)
- Ensure smooth playback, no stuttering

## Video Player Implementation (Landing Page)

### HTML5 Video Element:

```html
<video
  autoplay
  muted
  loop
  playsinline
  poster="/videos/hero-video-poster.jpg"
  className="w-full rounded-xl shadow-2xl"
>
  <source src="/videos/hero-video.mp4" type="video/mp4">
  <source src="/videos/hero-video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>
```

**Attributes:**
- `autoplay muted` - Autoplays without sound (allowed by browsers)
- `loop` - Seamless repeat (OR add replay button)
- `playsinline` - Prevents fullscreen on iOS
- `poster` - Image shown before video loads

**Custom Controls (Optional):**
- Play/Pause button overlay
- Mute/Unmute toggle
- Replay button (if not looping)
- Progress bar
- Fullscreen button

**Lazy Loading:**
- Only load video when hero section is in viewport
- Use Intersection Observer API
- Saves bandwidth for users who don't scroll to hero

## Accessibility Considerations

### Video Accessibility:
- **Captions/Subtitles:** If on-screen text appears, ensure it's readable (high contrast, large enough)
- **Alt Text:** Provide text description of video for screen readers
  - "Video showcasing 3 process templates: lead conversion with AI scoring, order management with quality alerts, and customer success with churn prediction"
- **No autoplay with sound:** Muted autoplay is okay, audio autoplay is not
- **Pause control:** Users can pause anytime (keyboard accessible)

### Testing:
- Test with keyboard navigation (Space to play/pause)
- Test with screen readers (VoiceOver, NVDA)
- Test on slow connections (video should not block page load)

---

## Performance Optimization

### Video Loading Strategy:
1. **Poster image loads first** (instant, <50KB)
2. **Video lazy loads** when hero section in viewport
3. **Preconnect to CDN** for faster video fetch
4. **Below-fold content loads after** video (priority to hero)

### Fallback:
- If video fails to load: Show static poster image with "Play Video" button
- If JavaScript disabled: Show static hero image with key message
- If slow connection: Detect and offer "Play Video" button (don't auto-load)
