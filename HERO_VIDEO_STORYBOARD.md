# Hero Video Storyboard (Option B: Dashboard Walkthrough)

## Video Specifications

**Duration:** 35-40 seconds
**Format:** MP4 (H.264), WebM (VP9) for web
**Resolution:** 1920×1080 (Full HD) - Source in 4K if possible
**Aspect Ratio:** 16:9
**Frame Rate:** 30fps or 60fps
**Audio:** Subtle background music (optional, no voiceover)
**Loop:** Seamless loop OR replay button
**File Size Target:** <5MB (optimized for web)

---

## Story Arc

**Theme:** Show a complete workflow transformation from lead capture to deal close, highlighting automation and intelligence at key moments.

**Narrative:** A business receives a lead, CoreShift automatically routes it, tracks activity, provides AI insights, and closes the deal — all in one seamless flow.

## Visual Design Guidelines

### UI Aesthetic:
- **Match landing page design system:**
  - Glassmorphism effects (subtle backdrop blur)
  - Gradient accents (purple/blue theme)
  - Clean typography (sans-serif, high readability)
  - Rounded corners (consistent border radius)
  - Subtle shadows (elevation system)

### Color Palette:
- **Primary:** Blues, purples (matching CoreShift brand)
- **Success:** Green (for completed tasks, won deals)
- **Neutral:** Grays (backgrounds, secondary text)
- **Accent:** Bright colors for key actions (CTAs, highlights)

### Animations:
- **Smooth, professional** (not bouncy or cartoonish)
- **Ease-out transitions** (natural movement)
- **Subtle micro-interactions** (hover states, clicks)
- **Avoid:** Jarring cuts, fast movements, distracting effects

### Typography:
- **Readable at all sizes** (video will be scaled down)
- **High contrast** (dark text on light backgrounds, or vice versa)
- **Consistent hierarchy** (headlines, body, labels)

## Recording & Production Approach

### Recommended Workflow:

#### 1. Build Mockup Dashboard (Week 1-2)
- Create functional Next.js prototype in `coreshift-mockups` project
- Implement all screens shown in storyboard:
  - Dashboard home/inbox
  - Lead detail page with activity timeline
  - Pipeline Kanban board
  - CompassAI insight panel
  - Success/metrics view
- Use realistic sample data (names, companies, numbers)
- Ensure all animations work smoothly

#### 2. Screen Recording (Week 3)
**Tools:**
- **Mac:** Screen Studio (https://screen.studio) - PERFECT for this
- **Windows:** OBS Studio or Camtasia
- **Cross-platform:** Loom (simpler but less polished)

**Recommendation:** Use **Screen Studio** (since you referenced it)
- Produces beautiful, polished screen recordings
- Auto-smooths cursor movement
- Adds professional zoom/pan effects
- Exports high-quality MP4

**Recording Tips:**
- Record at 4K resolution (downscale to 1080p for final)
- Use 60fps for smoother animations
- Rehearse the workflow 3-4 times before final recording
- Record in segments if needed (easier to edit)
- Keep cursor movements intentional and smooth

#### 3. Post-Production (Week 3)
**Video Editing:**
- **Tools:** Final Cut Pro, Adobe Premiere, DaVinci Resolve (free), iMovie (simple)
- Trim and polish recordings
- Add on-screen text labels (After Effects or editor's built-in titles)
- Add ShiftIQ/CompassAI badges at key moments
- Color grade for consistency
- Add background music and sound effects (subtle)

**Motion Graphics (Optional Enhancement):**
- Add animated badges/icons for ShiftIQ and CompassAI
- Enhance transitions between shots
- Add subtle particles/glow effects for "magic moments"
- Tools: After Effects, Motion (Mac)

#### 4. Optimization for Web (Week 3)
- Export at 1920×1080, 30fps, H.264 codec
- Target bitrate: 5-8 Mbps (balance quality/file size)
- Create WebM version for modern browsers (better compression)
- Compress with Handbrake or FFmpeg if file size >5MB
- Test autoplay and loop functionality
- Create poster image (thumbnail for before video loads)

## Video Player Implementation on Landing Page

### Technical Specs:

**HTML5 Video Element:**
```html
<video
  autoplay
  muted
  loop
  playsinline
  poster="/hero-video-poster.jpg"
>
  <source src="/hero-video.mp4" type="video/mp4">
  <source src="/hero-video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>
```

**Attributes:**
- `autoplay muted` - Autoplays without sound (allowed by browsers)
- `loop` - Seamless repeat
- `playsinline` - Prevents fullscreen on iOS
- `poster` - Image shown before video loads

**Custom Controls:**
- Play/Pause button overlay
- Mute/Unmute toggle
- Replay button (if not looping)
- Optional: Fullscreen button

**Lazy Loading:**
- Only load video when hero section is in viewport
- Use Intersection Observer API
- Saves bandwidth for users who don't scroll to hero

## Accessibility Considerations

### Video Accessibility:
- **Captions/Subtitles:** If text labels appear, ensure they're readable (high contrast)
- **Alt Text:** Provide text description of video for screen readers
- **No autoplay with sound:** Muted autoplay is okay, audio autoplay is not
- **Pause control:** Users can pause anytime (keyboard accessible)

### Testing:
- Test with keyboard navigation (Space to play/pause)
- Test with screen readers (VoiceOver, NVDA)
- Test on slow connections (video should not block page load)

---

## Performance Optimization

### Video Loading Strategy:
1. **Above-the-fold priority:** Hero video loads first
2. **Lazy load other images:** Screenshot rows load as user scrolls
3. **Preload poster image:** Shows immediately while video buffers
4. **CDN delivery:** Host video on Vercel/Cloudflare CDN for fast delivery

### Fallback:
- If video fails to load: Show static poster image with "Play Video" button
- If JavaScript disabled: Show static hero image
